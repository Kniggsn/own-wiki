Basic Syntax

Selectors

Universal selectors

For example: * {color: purple;}

Selects elements of any type - just a simple "*"

Type Selectors

for example: div {}

Selects all elements from the given element type

Class Selectors

For example: .class-name {}

A period followed by the case-sensitive value of the class attribute.

Selects all elements with the given class

To add a class to an HTML-element, just add the attribute "class="class-name""

ID Selectors

For example: #title {}

A hashtag followed by the case-sensitive value of the class attribute.

To add an ID to an HTML-element --> attribute id="ID-name"

Don't overuse! Mostly classes will suffice. One use case is to redirect to a section on the current page.

Grouping Selectors

For example:

.read,

.unread {color:white;}

Now read and unread have the color white --> after that you can give .read and .unread additional unique declarations.

Chaining Selectors

Give a html-Element multiple classes by dividing them with a space, for example: class="subsection header" --> belongs to the classes "subsection" and "header"

Now if we only want to give a rule to an element that has the subsection-class as well as the "header"-class, we would have to define our CSS as ".subsection.header {}"

Works for chaining any combination of selectors, except for more than one type selector. To chain class and ID-selectors you would have to write: .subsection#preview

Descendant Combinator

.ancestor .child {}

This would choose any child-class that has a .ancestor class as parent (no matter how deep .child is nested relatively to .ancestor

Child combinator >

.class > . second-class{}

second-class needs to be the immediate child of class

Basic Properties

Color and Background-Color

  • per Hex: color: #1100ff
  • Hex with Transparency: #1100ff80 --> add two additional digits between 00 and FF
  • per rgb: background-color: rgb(100, 0 127)
  • per rgb alpha: rgba(255, 0, 0, 0.3) --> number between 0.0 (fully transparent) and 1.0 for alpha
  • per hsl: color: hsl(15, 82%, 56%)
  • per hsl alpha hsla(15, 82%, 56%, 0.3
  • predefined: there are 140 predefined colors in html and CSS (e.g. blue, red, coral, ...) see here.

Typography Basics and Text-Align

  • font-family --> single value or comma separated list. If first font can't be found, the second one is used etc. Either the font gets a "font family name" like "DejaVu Sans" or a "generic family name" like sans-serif. There should always be a generic backup font.
  • font-size --> e.g. font-size: 22px
  • font-weight --> number between 1 and 1000 --> boldness (700 = bold), usually 100 up to 900 can be used
  • text-align --> start, end, center or justify and more; see here.

Image Height and Width

For example height: auto; width: 500 px. That way the image keeps it proportions.

Should always be included --> so the image takes it's space even if it needs some time to load.

The Cascade of CSS

Specificity

A CSS declaration that is more specific will take precedence over less specific ones.

  1. ID selectors (most specific)
  2. Class selectors
  3. Type selectors

Inheritance

Children of an Element inherit it's properties, except when they are targeted directly

Rule Order

When no rule applies - the last defined element is the winner.

Adding CSS to HTML

External CSS

Most common - separate CSS file. Has to be linked inside the HTML's opening and closing head-tags with self-closing link-element (link rel="stylesheet" href="styles.css")

href --> absolute or relative link; rel --> defines the relationship between the HTML and linked file.

--> keeps HTML and CSS separated --> cleaner; only one place to edit CSS - handy for websites with many pages in similar styles

Internal CSS

Adding CSS within the HTML-file --> Style-Element inside the head element with all the CSS for the page

Can be useful for adding unique styles to a single page.

Inline CSS

Add styles directly to HTML-Elements - not as recommended. Done by adding a style attribute to a element with all CSS-declarations separated by a semicolon. Selectors don't work.

For unique style of a single element this might work However:.

  • Gets messy with a lot of declarations.
  • If a lot of elements have the same style --> lots of copy paste.
  • Will override the other two methods (this can however be taken advantage of.).
