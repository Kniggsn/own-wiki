Javascript

The Basics

How to run Javascript-Code

  • Via the browser --> just use the "script" Element
  • Via an external script --> like with CSS files --> use the Script Element and link the Javascript file with the src-attribute

Variables

  • basically "storage containers" for data
  • until recently the key word "var" was used to create a variable
  • the new way to create variables it the "let" keyword, e.g. "let message" or even "let message = 'hello world!'"
  • the new way to create a constant is the "const" keyword
  • It's possible to declare multiple variables in one line - it is however not recommended
  • There is a slight difference between var and let - just use let for now
  • A variable can only be declared one - using the "let" keyword on an already existing variable will cause an error

Naming the variable

  • camelCase is widely used
  • name must contain only letters, digits, $ or _
  • first character must not be a digit
  • case sensitive
  • there are reserved names that can't be used (e.g. let, return, ...)
  • constants that ar aliases for difficulty to remember values --> capital letters and underscore, e.g. "COLOR_RED"

Numbers

  • only one type of number in JS --> can be written with or without decimals
  • numbers are always 64bit floating point
  • Other types of number would be:
    • Integers --> floating-point without a fraction - negative or positive
    • Floats --> floating point numbers - have a decimal point
    • Doubles --> Special type of float with higher precision
  • very large or small numbers can be written with exponent notation --> e.g. 123e5 (=123*10‚Åµ)
  • integers are accurate up to 15 digits
  • maximum number of decimals is 17
  • floating point arithmethic is not always 100 % accurate --> 0.2 + 0.1 = 0.30000000000000004
  • to solve this it can help to multiply and divide (0,2+0,1)*10/10 = 0,3
  • if a number and a string is added - the result will be a string concatenation - 10 + "20" = "1020" or 20 + 40 + "6" = "606"
  • JS will however try to convert strings with numeric content --> "10" * "20" = 200 --> works with everthing besides "+"
  • Trying to do arithmetic with a non-numeric string will result in "NaN" (Not a Number)
  • Infinity --> will be returned if a number is calculated outside the largest possible number or by division by zero
  • number that starts with "0x" is interpreted as hexadecimal --> 0xFF = 255
  • by default numbers are displayed as base 10 decimals - use toString(16) for output as hexadecimal (or another number - highest possible is 32)

Arithmethic Operators"

  • + - Addition
  • - - Substraction
  • * - Multiplication
  • ** - Exponentiation
  • / - Division
  • % - Remainder (Modulus)
  • ++ - Increment
  • ++variable --> increments variable and then returns the incremented variable
  • variable++ --> returns the variable and then increments it by one
  • -- - Decrement
  • + is the operator - the number for example is the operand

Build in Functions and Methods

  • console.log() --> print something into the developer console
  • alert(message) --> creates a popup with the message
  • isNaN(x) --> checks of x is NaN, returns bool
  • typeof NaN --> is a operator --> returns a string of the type of the operand's values --> in this example "number"
  • number.toFixed(2) --> rounds an value of type number to two decimal places
  • Number(string) --> convert String to number
  • number.toString --> Method of number, converts it to string

Operators

  • unary --> an operator with a single operand (e.g. -1)
  • binary --> an operator with two operands
  • === - strict equality
  • !== - Strict non euality
  • == - non-strict quality (checks if value identical - but not if datatype --> not recommended)
  • < or> - less or more than
  • <= or>= - Greater/Less than or qual to
  • unary + - e.g. +true --> converts to a number, in this case 1 (same as Number())

Bitwise Operators

  • AND (&)
  • OR (|)
  • XOR (^)
  • some more
  • used rarely when needed to fiddle with the numbers

Datatypes

  • Strings
    • basically a piece of text
    • To signal the browser that the datatype is a string, you have to put the value into single or double quotes (stick to one of them!)
    • To use a single quote inside of an single-quoted string you have to escape the single quote with a backslash
    • You can put variables in your string with ${variable}

    String methods and properties

    • string.length --> length of the string
    • string.slice(start-index, end-index) --> returns slice of original string (end-index not included)
    • string.slice(start) --> returns sliced string (start index until end)
    • string.slice(-12) --> negativ indices are counted from the end
    • string.substring(start, end) - same as slice - differenc: negativ numbers are treated as 0
    • string.substr(start, length) --> if length is not specified, the string from the start index on will be returned
    • string.replace("text to be replaced", "new text") - replaces text --> does NOT replace the original string but returns a new string; only the first match is replaced
    • string.replaceAll(old text, new text) - if a RegEx is used here, the g flag has to be set
    • RegEx can be specified in the following form: /pattern/modifiers - without quotes - also see here
    • toLowerCase(); toUpperCase() --> returns string as expected
    • text1.concat(" ", text2) --> "text1 text2"
    • text.trim() --> removes whitespaces from both sides of the string (there is also trimStart(), trimEnd()
    • text.padStart(6, "x") --> adds 6 times an x on the left side of the string
    • text.charAt(index) --> returns char at index (if not found --> empty string)
    • text[index] --> basically same, but returns undefined if not found (read only!)
    • text.charCodeAt(index) --> returns unicode of character
    • text.split("separator") --> returns an array of the string, splitted at the specified separator
    • See here for some more String methods and properties

Conditionals

  • All comparisons return a boolean value (true or false)
  • String comparison --> done by lexicographical order (B > A; strings are compared letter by letter)
  • comparison of different types --> e.g. "2" > 1 --> true - JS converts "2" to 2
  • mathematical comparisons --> null gets converted to 0; undefined to NaN
  • but equality check (==) --> null won't be converted
  • undefined = null --> true - any other comparison with undefined will return false
  • if, else, else if --> checks if a conditions is true and does something else if not
  • switch(expression) {case: choice a: code; break; ... default: code} --> checks every defined case - default is run if no case is truthy
  • Ternary operator --> condition ? run this code if true : else run this code --> smaller than if...else - should be used to return one value or another (e.g. when asigning a value to a const)--> can also be chained!
  • || --> OR; && --> AND, ! --> NOT, ?? --> Nullish Coalescing
    • OR --> returns first truthy value (true, 1, ...) or the last if none is found (e.g. false, 1, undefined)
    • AND --> finds first falsy value, returns last one if none is found; has a higher precedence than OR
    • ! --> Converts operand to boolean and than inverts it - a !! can be used to convert to boolean (better: use Boolean()) - has highest precedence

Manipulating the DOM

DOM-Manipulation

  • Target Nodes:
    • div.display --> div element with display class
    • .display --> every element with display class
    • #container > .display --> an element with display-class that is inside an element with the ID "cointainer"
    • container.firstElementChild --> selects the first Element Child of container
  • DOM Methods
    • element.querySelector(selector) --> returns reference to first match of selector
    • element.querySelectorAll(selectors) --> returns a "nodelist" (not exactly an array) containing references to all the matches of the selectors
    • document.createElement(tagName, [options]) --> creates new element of tagType tagName e.g. const div = document.createElement("div"); --> does not put the element into the dom yet!
    • parentNode.appendChild(childNode) --> appends childNode at the last child of the parent
    • parentNode.insertBefore(newNode, referenceNode) --> inserts new node into parent node before the reference
    • parentNode.removeChild(child) --> removes child from parentNode on the DOM and returns a reference to child
    • nodeReference.stlye.color = blue --> adds the indicated style rule
    • nodeReference.style.cssText = "color: blue; background: white"
    • nodeReference.setAttribute("style", "color: blue; background: white;")
    • Note that for kebab-cased CSS rules from JS you'll need to use camelCase or bracket notation, e.g. nodeReference.style.background-color does not word; .backgroundColor and style["background-color"] or even .cssText = "background-color: white" does work.
    • nodeReference.setAttribute("id", "theDiv") --> if "id" exists, update it to theDiv or create an id with value div
    • referenceNode.getAttribute("id")
    • referenceNode.removeAttribute("id")
    • referenceNode.classList.add("new") --> adds class "new" to referenceNode
    • referenceNode.classList.remove("new")
    • referenceNode.classList.toggle("new")
    • referenceNode.textContent = "Hello World"
    • div.innerHTLM = "Hello World"
