Flexbox

What is flexbox?

Flexbox is a relatively new way of manipulating elements in CSS. It's used to position elements with so called flex containers and flex items.

The Basics

For "flexboxing" it is important to always see elements as containers and the items of the container.

You see, that the p elements inside a Container with flex don't start a new line anymore despite being block-Elements.
So long story short: A flex container is any element that has display: flex as a property.
A flex item is any element inside a flex-container.

Flex items can however also be flex containers!

Modern websites mostly use flexbox. However, older websites used so called Floats.

Aligning Items

Aligning items with Flexbox - Flex-Containers

After giving the flex container the "flex" property, you can tell the container how to align it's content.
  • justify content: center/flex-start/flex-end - aligns the items as chosen
  • justify content: space-around/space between - useful with multiple items --> give the container a width and these properties will position the items on the page equally (either with the came space at the left and right of the outer elements or without)
  • align-items - can be used for vertical alignment if the container has a specified height See this example image:
  • flex-wrap: wrap/nowrap - with wrap activated items will automatically start a new line, if they have a defined width and there are too many items to be fit in one line of the container
  • flex-direction: row/column/row-reverse/column-reverse - renders items horizontally or vertically - justify-content changes the vertical alignment when the flex-direction is column

Aligning items with Flexbox - Flex-Items

It's also possible to manipulate individual items of a flex-container.
  • order: 0/-1/1/... - positions the item at the defined position
  • align-self: flex-end - positions the item at the bottom of the container
  • flex: 1/2/3/... - makes items expand it's width - if all items have a flex of 1, they will fill up their container and all have an equal width - items with a higher with will grow bigger than the other items
  • you can also define static item widths --> flex: initial --> these items will keep their defined width - should also work by just not defining the "flex" property for the items that should not be responsive
  • flex is the shorthand for flex-grow, flex-shrink and flex-basis - when setting flex: 0 it is the shorthand for flex: 0 0 1
  • flex-grow --> if we give two items a value of 1 and one item the value of 2, the item with the value of two will be twice as big as the other items
  • flex-shrink --> same as grow, just with shrinking
  • baseline: auto --> if we have a set width for items, they will try to get that width first and after that rules like "flex-grow" are applied if possible - shrink is applied if the items don't have enough space when shrinking the view (so the item with a higher value for shrinking will be smaller) - growing is applied if there is enough space for the items to fill there set value for width
  • flex: initial - is equivalent to flex: 0 1 auto --> sizes based on width/height - so inflexible when there is free space, but allows to shrink
  • flex: auto - is equivalent to flex: 1 1 auto --> sizes based on width/heigth, but makes items fully flexible --> absorb any space along the main axis
  • flex: none - is equivalent to flex: 1 1 0 --> not flexible
  • flex: 3 - is equivalent to flex: 4 1 0
  • min-width/min-height - stes the items minimum width/height --> flex items can shrink below their minimum content size
  • It's often easier to see the shorthand property as "flex: [max] [min] [ideal size]
  • If I have for example three flexitems with one of them having flex: 1 but there is much more content in one of the items with flex: 1, then the flex: 3 item won't be the biggest
