The Box Model

The box model is used to put your Elements exactly where you want them to be.

The Basics

You can use margin, padding and borders to make sure elements are just the right size.
To get a grasp of how the box model works, you basically have to imagine every single HTML-element as a box. You can actually make these boxes visible with CSS by entering the code * {border: 2px solid red;}. However - the above mentioned elements work as follows:

  • padding - increase space between border of a box and content of box
  • margin - increase space between borders of box and adjacent box. Can also be negative for overlapping.
  • margin between two objects is not summed up - instead the bigger one is chosen as space (only for vertical)(if one is negative and one is positive the negative one will be substracted from the total.
  • margin: margin top right bottom left --> to set e.g. 4 different margins
  • if width is set to e.g. 980px and margin to "0 auto" --> specified width - left and right margins are automatically set
  • border - adds space between margin and padding
  • outline --> A outline border is added around the actual border of an element to make it stand out.
  • box-sizing: border-box - adjusts height/width of the actual content so that this size + padding + border are the size that's set in the width/height property - also called "alternative box model"

Block and inline boxes

There are several types of boxes. The type refers to how they behave in terms of page flow and in relation to other boxes. Boxes have an inner and outer display type.

Outer display type

If the outer display type is "block", then:

  • box will break onto new line
  • width and height are respected
  • by default uses all space available (from left to right)
  • padding, margin and border will cause other elements to be pushed away from the box
  • if width is not specified --> box will extend in inline direction to fill space available
  • some elements like h1, p use block by default
  • by default block elements ar laid out in the block-flow-direction, which is based on the parents writing-mode
  • default writing-mode is horizontal-tb (top to bottom)

If the outer display type is "inline", then:

  • box will not break onto new line
  • width and height will not apply (exception: img-Element)
  • vertical padding, margins and borders will apply, but not cause other inline boxes to move away from the box
  • horizontal padding etc. will apply and will cause other inline boxes to move away
  • in general you don't want to use padding or margin on inline elements
  • default for a, span, em, strong, ...

Inner display type

  • defines how elements inside the box are laid out
  • you can change the type in CSS with the property "display: inline"

Using display: inline-block

  • display: inline-block
  • special value of display - middleground between inline and block
  • width and height are respected and padding etc. cause other elements to be pushed away from the box
  • does not break onto a new line
  • better alternative than inline-block is mostly flexbox
